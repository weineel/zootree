# GitHub CI/CD Release Design

## Overview

Automate the release pipeline for `zootree` using `cargo-dist` and `cargo-release`.
A single command (`cargo release minor`) handles version bumping locally; CI handles
cross-platform builds, GitHub Release creation, Homebrew tap updates, and crates.io publishing.

## Tools & Responsibilities

| Tool | Responsibility |
|------|---------------|
| `cargo-release` | Local: bump version → commit → tag → push |
| `cargo-dist` | CI: cross-platform builds → GitHub Release → Homebrew tap → crates.io |
| `ci.yml` (handwritten) | Every push/PR: fmt + clippy + test |
| `release.yml` (generated) | Every `v*` tag: check → build → publish |

## Release Flow

```
cargo release minor
  → bumps Cargo.toml version, commits, creates tag v0.x.0, pushes
    → release.yml triggered
      → check job (tests pass)
        → build job (5 platforms in parallel)
          → publish job (GitHub Release + Homebrew tap + crates.io)
```

## Build Targets

| Target | Platform |
|--------|----------|
| `x86_64-apple-darwin` | macOS Intel |
| `aarch64-apple-darwin` | macOS Apple Silicon |
| `x86_64-unknown-linux-gnu` | Linux x86_64 |
| `aarch64-unknown-linux-gnu` | Linux ARM64 |
| `x86_64-pc-windows-msvc` | Windows x86_64 |

## Configuration

### `Cargo.toml` additions

```toml
[workspace.metadata.dist]
cargo-dist-version = "0.28.0"
ci = ["github"]
targets = [
  "x86_64-apple-darwin",
  "aarch64-apple-darwin",
  "x86_64-unknown-linux-gnu",
  "aarch64-unknown-linux-gnu",
  "x86_64-pc-windows-msvc",
]
installers = ["shell", "homebrew"]
tap = "weineel/homebrew-tap"
publish-jobs = ["crates-io"]
```

### `release.toml` (new file, project root)

```toml
sign-commit = false
sign-tag = false
push = true
publish = false
tag = true
pre-release-commit-message = "chore: release {{version}}"
tag-message = "v{{version}}"
```

## GitHub Workflows

### `ci.yml` (handwritten)

Triggered on every push to `main` and every PR.

```yaml
on:
  push:
    branches: [main]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: clippy, rustfmt
      - run: cargo fmt --check
      - run: cargo clippy -- -D warnings
      - run: cargo test
```

### `release.yml` (auto-generated by `cargo dist init`)

Triggered on `v*` tag pushes. Structure:
- **check job**: runs tests as a quality gate
- **build job**: matrix across 5 targets, produces binaries + shell installer
- **publish job**: creates GitHub Release with auto-generated release notes,
  updates `weineel/homebrew-tap` Formula, publishes to crates.io

## Installers

- **GitHub Release**: pre-built binaries attached to each release
- **Shell script** (`curl | sh`): installs the correct binary for macOS/Linux
- **Homebrew tap**: `brew install weineel/tap/zootree`

## Release Notes

GitHub's built-in auto-generate from commits/PRs between tags. No manual CHANGELOG maintenance.

## Prerequisites

Before the first release:

1. Create GitHub repo `weineel/homebrew-tap` (can be empty)
2. Set GitHub Secrets on `weineel/zootree`:
   - `CARGO_REGISTRY_TOKEN`: crates.io API token (from https://crates.io/settings/tokens)
   - `HOMEBREW_TAP_TOKEN`: GitHub PAT with write access to `weineel/homebrew-tap`
3. Install tools locally:
   ```bash
   cargo install cargo-dist
   cargo install cargo-release
   ```
4. Run `cargo dist init` to generate `release.yml` and update `Cargo.toml`

## Developer Workflow

```bash
# Patch release (0.1.0 → 0.1.1)
cargo release patch

# Minor release (0.1.0 → 0.2.0)
cargo release minor

# Major release (0.1.0 → 1.0.0)
cargo release major
```
